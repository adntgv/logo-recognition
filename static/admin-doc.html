Обзор системы

Что это такое?

Система распознавания логотипов HalalDamu — это автоматизированный сервис для идентификации халал-сертифицированных компаний по их логотипам на фотографиях продуктов.

Зачем это нужно?

Потребители могут быстро проверить, является ли продукт халяльным, просто сфотографировав логотип производителя. Система мгновенно определяет компанию и подтверждает её статус сертификации в базе HalalDamu.

Как это работает?

Система использует современные технологии компьютерного зрения и машинного обучения:

  • Визуальное распознавание: нейросеть CLIP (ViT-B-32) извлекает визуальные признаки из логотипа
  • Многократная обрезка: изображение анализируется в нескольких масштабах и позициях для повышения точности
  • Поиск по индексу FAISS: быстрый поиск похожих логотипов в базе эталонов
  • OCR (распознавание текста): дополнительная проверка названий компаний на изображении
  • Гибридная оценка: комбинация визуального сходства (85%) и текстового совпадения (15%)
Технические характеристики:
  • Модель: OpenCLIP ViT-B-32 (обучена на 2B изображений)
  • Индекс: FAISS (быстрый поиск по векторам)
  • Среднее время распознавания: менее 1 секунды
  • Точность на размеченных данных: отображается в панели статистики

Для пользователей

Как загрузить фото?

  1. Откройте главную страницу сервиса
  2. Нажмите на область загрузки или перетащите фото логотипа
  3. Дождитесь результата (обычно менее 1 секунды)

Что означают результаты?

Система возвращает список до 10 наиболее похожих компаний с оценками:

  • Название компании: официальное название из базы HalalDamu
  • Visual Score: степень визуального сходства логотипа (0-1)
  • Text Score: степень совпадения текста на изображении с названием компании (0-1)
  • Final Score: итоговая оценка (комбинация визуального и текстового скоров)

Что значит "не найдено"?

Сообщение "unknown" (не найдено) появляется в следующих случаях:

  • Логотип не похож ни на один эталон в базе (Final Score < 0.3)
  • Компания не сертифицирована HalalDamu
  • Фото низкого качества или логотип частично скрыт
  • На изображении нет логотипа
Важно: отсутствие компании в результатах не означает, что продукт не халяльный. Возможно, компания ещё не добавлена в базу или логотип на фото не распознан корректно.

Разметка фото

Разметка позволяет улучшать точность системы, подтверждая правильные предсказания и исправляя ошибки.

Как размечать фото

Для каждой фотографии доступны следующие действия:

1. Подтвердить (Confirmed)

Если верхний результат (с наивысшим score) правильный:

  • Нажмите на кнопку с названием компании (первая кнопка с синей рамкой)
  • Статус изменится на "confirmed"
  • Эта разметка улучшит точность будущих предсказаний

2. Исправить (Corrected)

Если правильная компания есть в списке, но не на первом месте:

  • Нажмите на кнопку с правильным названием компании (2-5 кнопки)
  • Статус изменится на "corrected"

Если правильной компании нет в предложенных вариантах:

  • Начните вводить название в поле "Search company..."
  • Появится выпадающий список совпадений
  • Выберите нужную компанию из списка

3. "Нет в базе HalalDamu" (Not HD)

Если компания на логотипе не сертифицирована HalalDamu:

  1. Нажмите красную кнопку "Not HD"
  2. В модальном окне введите название компании, которое видите на логотипе
  3. Нажмите "Submit"

Система:

  • Сохранит компанию в список несертифицированных
  • Пометит фото как "not_halaldamu"
  • Будет отслеживать частоту запросов для анализа

4. "Не знаю" (Unknown)

Если невозможно определить компанию (размытое фото, логотип не виден и т.д.):

  • Нажмите оранжевую кнопку с вопросительным знаком
  • Фото будет помечено как "unknown"
  • Оно не будет использоваться для обучения
Совет: старайтесь размечать как можно больше фотографий. Каждая размеченная фотография улучшает точность системы. Начните с вкладки "Unlabeled", где собраны все неразмеченные изображения.

Статистика

Верхняя панель отображает ключевые метрики системы в реальном времени:

Метрика Описание
Total Общее количество загруженных пользователями фотографий
Unlabeled Количество фото, ожидающих разметки
Confirmed Фото, где верхнее предсказание было подтверждено модератором
Corrected Фото, где модератор выбрал другую компанию из списка или через поиск
Unknown Фото, которые невозможно идентифицировать
Accuracy Точность системы: Confirmed / (Confirmed + Corrected) × 100%

Как читать Accuracy?

Accuracy показывает, как часто система правильно угадывает компанию с первой попытки:

  • 90-100% — отличная точность, система работает очень хорошо
  • 70-89% — хорошая точность, но есть пространство для улучшения
  • 50-69% — средняя точность, требуется пересборка индекса с учётом новых разметок
  • < 50% — низкая точность, необходима срочная доработка (больше эталонов, пересборка)
Важно: accuracy рассчитывается только по размеченным фото (confirmed + corrected). Unknown и not_halaldamu не учитываются, так как для них нет правильного ответа в базе.

Синхронизация компаний

Кнопка "Sync Companies"

Эта функция обновляет базу компаний из API HalalDamu.

Что делает синхронизация?

  1. Загружает актуальный список всех сертифицированных компаний из HalalDamu API
  2. Скачивает логотипы новых компаний
  3. Генерирует векторные представления (embeddings) для новых логотипов
  4. Обновляет FAISS индекс
  5. Сохраняет изменения в MinIO (облачное хранилище)
  6. Перезагружает индекс в оперативную память сервиса

Когда использовать?

  • Регулярно: раз в неделю или раз в месяц для получения новых компаний
  • После уведомления: если HalalDamu добавил новые сертификаты
  • При низкой точности: если пользователи часто получают "unknown" для известных брендов
Внимание: синхронизация может занять 2-5 минут в зависимости от количества новых компаний. На время синхронизации система остаётся доступной, но старые данные могут отдаваться до завершения процесса.

Результат синхронизации

После завершения появится уведомление с информацией:

  • Количество компаний в базе (total)
  • Количество новых логотипов (new_logos)
  • Количество векторов в индексе (reloaded_vectors)

Пересборка индекса

Кнопка "Rebuild Index"

Пересборка индекса включает размеченные пользовательские фото в обучающую выборку.

Когда нажимать?

  • После разметки 50+ фото: когда накопилось достаточно подтверждённых и исправленных примеров
  • При падении accuracy: если точность снизилась ниже 80%
  • Раз в неделю/месяц: как регулярная процедура поддержания качества

Что происходит?

  1. Скрипт rebuild_index.py запускается на сервере
  2. Загружаются все эталонные логотипы компаний
  3. Добавляются размеченные пользовательские фото (confirmed + corrected)
  4. Для всех изображений генерируются embeddings
  5. Строится новый FAISS индекс
  6. Сохраняется в index_artifacts/ и MinIO
  7. Индекс перезагружается в память
Результат: система начинает лучше распознавать логотипы, похожие на те, что уже были размечены. Accuracy обычно растёт на 5-15% после первой пересборки.
Важно: пересборка может занять 1-3 минуты. На время пересборки система продолжает работать со старым индексом. Перезагрузка происходит атомарно без downtime.

Несертифицированные компании

Кнопка "Unverified"

Открывает таблицу компаний, которые пользователи пытались найти, но которых нет в базе HalalDamu.

Как просмотреть список?

  1. Нажмите оранжевую кнопку "Unverified" в верхней панели
  2. Появится таблица с топ-50 компаниями
  3. Повторное нажатие скрывает таблицу

Столбцы таблицы

Столбец Описание
Name Название компании, введённое модератором
Requests Сколько раз модераторы помечали фото этой компании
First Seen Дата первого упоминания
Last Seen Дата последнего упоминания
Added By Логин модератора, который впервые добавил компанию

Зачем он нужен?

Этот список помогает:

  • Выявить популярные бренды: какие компании часто ищут пользователи, но нет в базе
  • Планировать сертификацию: передать данные команде HalalDamu для приоритизации
  • Анализировать тренды: понять, какие категории продуктов интересуют пользователей
  • Контроль качества: проверить, не дублируются ли названия (опечатки, разные написания)
Рекомендация: еженедельно экспортируйте топ-10 компаний из этого списка и передавайте команде сертификации HalalDamu для рассмотрения.

API эндпоинты

Краткое описание всех эндпоинтов для разработчиков и интеграций.

Публичные эндпоинты

POST /match

Распознавание логотипа на изображении.

curl -X POST -F "file=@logo.jpg" https://logo.adntgv.com/match

Ответ:

{
  "label": "Company Name",
  "candidates": [
    {
      "company_id": "123",
      "company_name": "Company Name",
      "visual_score": 0.9234,
      "text_score": 0.6500,
      "final_score": 0.8824
    },
    ...
  ],
  "ocr_tokens": ["Company", "Name"],
  "timing_ms": {
    "preprocess": 45,
    "embed": 234,
    "search": 12,
    "ocr": 187,
    "total": 478
  }
}

Эндпоинты авторизации

POST /api/login

Вход в систему для модераторов.

curl -X POST -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"halaldamu2026"}' \
  https://logo.adntgv.com/api/login

Ответ: {"token": "eyJ0..."}

Защищённые эндпоинты (требуется токен)

Все следующие запросы требуют заголовок: Authorization: Bearer <token>

GET /api/admin/stats

Получить статистику системы.

{
  "total": 1234,
  "unlabeled": 567,
  "confirmed": 420,
  "corrected": 89,
  "unknown": 123,
  "not_halaldamu": 35,
  "accuracy": 82.5
}

GET /api/admin/uploads

Список загруженных фото с пагинацией.

Параметры:

  • status — фильтр по статусу (unlabeled, confirmed, corrected, unknown, not_halaldamu)
  • page — номер страницы (по умолчанию 1)
  • limit — количество на странице (по умолчанию 20)

GET /api/admin/uploads/{upload_id}

Детальная информация о конкретном фото.

PATCH /api/admin/uploads/{upload_id}

Разметить фото.

curl -X PATCH -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"label_company_id":"123","label_status":"confirmed"}' \
  https://logo.adntgv.com/api/admin/uploads/456

GET /api/admin/companies?q={query}

Поиск компаний по названию (для autocomplete).

POST /api/sync

Запустить синхронизацию компаний с HalalDamu API.

POST /api/admin/rebuild-index

Пересобрать FAISS индекс с учётом размеченных фото.

GET /api/admin/unverified

Получить список несертифицированных компаний.

POST /api/admin/unverified

Добавить несертифицированную компанию.

curl -X POST -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"name":"New Company"}' \
  https://logo.adntgv.com/api/admin/unverified

GET /api/uploads/{upload_id}/image

Получить изображение загруженного фото (JPEG).

Документация API: для автоматической интеграции используйте Swagger UI по адресу /docs (если включено) или изучите исходный код app.py.
